Como Utilizar

A primeira coisa temos que criar e adicionar um arquivo javascript que contém os métodos necessários ao index do website: 
/**
 * ****************************************************
 * @Copyright (c) 2017, Spell Master.
 * @Version: 3.0
 * @Requisitos: Navegador compatível com HTML 5
 * ****************************************************
 * @class Gerencia APP TAB.
 * ****************************************************
 * @changes:
 * * 2.0 Spell Master
 * Modificado a captura do elemebto tab para data-* evt
 * * 3.0 Spell Master
 * - Alterado todo o comportamento, não precisando mais
 * vincular por parâmetros qual tab deve ser aberta.
 * Altomaticamente detecta o índice do botão e abre
 * o conteúdo pegando esse índice.
 * - Adicionado parâmetro opcional para poder gerenciar
 * as funções em diferentes locais do html.
 * - Adicionado suporte a funções por meio de ativação
 * externa.
 * ****************************************************
 * * @param propriety : (opcional) elemento para
 * determinar o conjunto de ativação.
 * ****************************************************
 */

var TabPaginator = function (propriety) {

    var node = propriety ? document.getElementById(propriety) : document;

    var tabLink = node.getElementsByClassName('tab-link');
    var tabBody = node.getElementsByClassName('tab-body');
    var i;

    this.getTab = getTab;

    /**
     * ************************************************
     * @function Abre a tab solicitada
     * @param index : índice do elemento class
     * ************************************************
     */
    function getTab(index) {
        return function () {
            openTab(index);
        };
    }

    /**
     * ************************************************
     * @function Abre a tab solicitada
     * @param tab : índice da tab para abrir
     * ************************************************
     */
    function openTab(tab) {
        for (i = 0; i < tabLink.length; i++) {
            tabLink[i].classList.remove('active');
            tabBody[i].classList.remove('active');
        }
        tabLink[tab].classList.add('active');
        tabBody[tab].classList.add('active');
    }

    /**
     * ************************************************
     * @function Inicia os botões
     * ************************************************
     */
    this.tabButtons = function () {
        for (i = 0; i < tabLink.length; i++) {
            tabLink[i].addEventListener('click', getTab(i), false);
        }
    };

    /**
     * ************************************************
     * @function Abre a primeira tab
     * ************************************************
     */
    this.startTab = function () {
        tabLink[0].classList.add('active');
        tabBody[0].classList.add('active');
    };
};

Usando os métodos

Adicionando o objeto mesmo que tudo esteja de acodo a coisa parece não funcionar porque?

Porque em grande parte talvez tenhamos que ativar as funções por meio externo.

Então agora veremos como utilizar as funções

No exemplo aqui vou usar o objeto como mencionado acima. Porém todos eles podem ser acionamos por prototipos, através de funções alheias.
Iniciação o objeto 
var tab = new TabPaginator();
Isso faz com que os cada botão funcione para abrir sua devida TAB 
tab.tabButtons();
Isso vai acionar a TAB que você determinar.

É necessário um parâmetro que é o índice de qual tab você quer abrir.

Por exemplo se eu tiver 4 índices dentro do escopo quer dizer que a primeiro índice é o numero 0 (zero) e o quarto índice é o número 3 (três)

tab.getTab(0);

Isso faz com que no carregamento do html o primeiro índice da TAB já seja aberto

Usado junto (nunca devem ser usados juntos) e após o método getTab irá tornar o método anterior não funcional.

tab.startTab();

Dicas


Mais de um Controle

Caso queira/precise de executar a função em diferentes locais dentro do html, basta que que adicione um elemento #ID e coloque o escopo da estrutura dentro desse elemento identificador.

E indicamos esse elemento ID como parâmetro na declaração do objeto

<div id="exemplo">
    <ul class="tab-menu">
        <li><a class="tab-link">TAB</a></li>
    </ul>
    <div class="tab-body">Conteúdo da TAB</div>
</div>
<script>
    var tab = new TabPaginator('exemplo'); // Passando o controle da TAB para o elemento ID#exemplo
</script>
Então para mais de um conjunto de TAB basta que crie um novo objeto (de nome diferente) e informe por parâmetro qual é o elemento ID do novo conjunto
Nesse caso você estará alterando o comportamento da localização dos elementos de trabalho das funções.

Quando você não informar por aprâmetro qual elemento ID está a arquitetura as funções vão buscar em todo DOM pelos elementos de trabalho.

Sendo necessáriamente obrigatório sempre informar por parâmetro o conjunto de trabalho.

Caso venha somente a usar um único conjunto, não precisa de informar o elemento identificador do conjunto.


Responsivo

Por ser algo que pode variar em sua quantidade ou uso de forma aleatória, não desenvolvi um esquema CSS para padronizar a cituação do responsivo.

Sendo necessário intervenção do desenvolvedor personalizar conforme a necessidade o uso das TABs.

No caso aqui dessa página apenas usei o media-query para maximizar os campos dos botões e adicionar um borda quando a resolução máxima for até 600px.

Mas como disse fica a seu critério a melhor forma dessa personalização

@media (max-width: 600px) {
    .tab-menu > li {
        border-bottom: 1px solid #d9d9d9;
        width: 100%
    }
}
